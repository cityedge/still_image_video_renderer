# Still Image Video Renderer v1.0.0 仕様メモ

## 目的

静止画像、音源ファイル（MP3またはWAV）、任意の字幕SRTファイルを合成し、静止画を背景に音楽が鳴るMP4動画を作成する。SRTが指定されている場合は字幕を焼き込む。

既存の多数のバッチファイル運用を置き換えることが目的であり、単なるffmpegラッパーではなく、プリセット選択、ドロップ操作、レンダープレビュー、微調整、プリセット保存を統合する。

## 入力

- 画像: `.jpg`, `.jpeg`, `.png`, `.bmp`, `.webp`
- 音源: `.mp3`, `.wav`
- 字幕: `.srt`（任意）

## 自動検出

音源ファイルを読み込んだ場合、以下を自動検索する。

1. 同じフォルダの `<音源名>.srt`
2. 音源だけをドロップし、画像が明示ドロップされていない場合、同じフォルダの `<音源名>_title.png`

SRTファイルを読み込んだ場合、以下を自動検索する。

1. 同じフォルダの `<SRT名>.mp3` または `<SRT名>.wav`
2. 画像が明示ドロップされていない場合、同じフォルダの `<SRT名>_title.png`

## 出力

- MP4
- H.264 / libx264
- AAC 48kHz 384kbps
- 30fps
- yuv420p
- faststart
- 音声長に合わせて終了
- 出力ファイル名候補は `<音源名>.mp4`
- 重複時は `_001`, `_002` などを付ける

## MP4ジョブ

`MP4作成` ボタンを押した時点で、画像、音源、任意のSRT、出力先、字幕スタイルをジョブとして固定する。

SRTが未指定の場合は、ffmpegの `subtitles` フィルタを使わず、字幕なしで静止画と音源のみのMP4を作成する。その他のエンコード設定はSRT指定時と同じとする。
ffmpeg実行中のジョブは以後のUI状態を参照しない。
ユーザーは実行中ジョブを待たずに次の素材選択、プリセット調整、プレビュー、次のMP4作成を続けられる。
自動出力候補は、実行中ジョブの予約済み出力先も考慮して重複を避ける。

## プリセット

プリセットは単一モデル。

```json
{
  "id": "preset_id",
  "name": {
    "ja": "日本語表示名",
    "en": "English display name"
  },
  "tags": ["任意タグ"],
  "style": {
    "fontname": null,
    "fontsize": null,
    "alignment": 6,
    "margin_v": 90,
    "primary_color": null,
    "outline_color": null,
    "outline": null,
    "shadow": null,
    "bold": null
  }
}
```

`name` は `ja` / `en` の言語別表示名を持つ。片方は `null` を許し、現在のUI言語側の値がない場合はもう一方へフォールバックする。クイックパネルのスロット表示名 `label` も同じ形式を使う。

`null` の値は `force_style` に出さない。

例：

```json
{
  "fontname": null,
  "fontsize": null,
  "alignment": 6,
  "margin_v": 90
}
```

生成する `force_style`：

```text
alignment=6,MarginV=90
```

## パネル

- 4列×5行
- 2ページ
- ページ名は変更可能
- 初期状態ではページ1に通常用とショート用の主要プリセットを配置し、ページ2は空き
- 各スロットは全プリセットへのショートカット
- パネルにファイルをドロップすると、そのスロットのプリセットを適用し、ファイルを読み込む
- パネルは画面右側に配置する
- パネルにマウスオーバーするとプリセットのフルネームを表示する
- パネルでプリセットを選ぶと、全プリセットのドロップダウンも同期する

## UI

- 主要ラベルは日本語/Englishを切り替えできる
- 言語切替は入力ファイル欄の上に置く
- Alignmentは `6（中央上）` / `10（中央）` / `2（中央下）` の三択
- 字幕サイズには `-5`, `-1`, `+1`, `+5` の調整ボタンを持つ

## プレビュー

### レンダープレビュー

ffmpegで1枚PNGを生成する。
最終出力と同じ `subtitles` フィルター、同じ `force_style` を使うため、最終判断の基準にする。
画像未指定時は白い16:9背景を一時生成して使う。
SRT未指定時はプレビュー用字幕テキストを使う。

### 簡易プレビュー

Pillowで画像に仮字幕を描く。
最終レンダーとは一致しない。
値を変更している途中の目安として使う。
画像未指定時は白い16:9背景を使う。
SRT未指定時はプレビュー用字幕テキストを使う。

簡易フォントサイズの現行ルール：

- `fontsize` 指定あり: `fontsize × 画像高さ / 288 × preview_font_scale`
- `fontsize` 指定なし: `16 × 画像高さ / 288 × preview_font_scale`
- `preview_font_scale` 暫定値: `0.76`

## 初期プリセット

`presets_default.json` 参照。

初期配置では、ページ1に通常系、ページ2にショート系を置いている。
ただし、運用上はページ1に「よく使う通常用とショート用」、ページ2に「マイナーな設定」を置く可能性がある。

## 最小成功条件

v0.1系では、次のフローが通ることを成功条件とする。

1. 音源のみをパネルへドロップ。
2. `<音源名>.srt` と `<音源名>_title.png` が自動で読まれる。
3. パネルのプリセットが適用される。
4. ffmpegレンダープレビューが表示される。
5. MarginVやフォントを微調整できる。
6. レンダープレビュー更新で実表示を確認できる。
7. MP4作成をジョブとして投入し、作成中に次の作業を開始できる。
8. 必要なら現在設定を新規プリセットとして保存できる。
