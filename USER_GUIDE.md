# Still Image Video Renderer v1.0.0 User Guide

Still Image Video Renderer は、静止画、音源ファイル、任意のSRT字幕を組み合わせて MP4 動画を作成する Windows 向けローカルアプリです。SRTがある場合は字幕を焼き込み、SRTがない場合は静止画と音源だけで動画を作成します。

このアプリは ffmpeg / ffprobe のラッパーです。画面上で選択した素材、プリセット、字幕スタイルは、最終的に ffmpeg の入力、フィルタ、エンコード設定へ変換されます。従来のバッチファイル運用で行っていた「音声をいったん WAV 化し、長さを測り、その長さで静止画動画を生成する」流れを GUI 化することを目的にしています。

## できること

- 静止画と音源を読み込み、MP4 を作成できます。
- SRT字幕がある場合は、字幕焼き込み済み MP4 を作成できます。
- 音源と同じフォルダに同名の `.srt` がある場合、自動で読み込みます。
- SRTと同じフォルダに同名の音源、または `<SRTファイル名>_title.png` がある場合、自動で読み込みます。
- 音源だけを読み込んだ場合、同じフォルダの `<音源ファイル名>_title.png` を自動検索して画像として使えます。
- クイックパネルへ素材をドロップし、プリセット適用と素材読み込みを同時に行えます。
- ffmpeg によるレンダープレビューを作成できます。
- 画像や SRT が未指定でも、白い 16:9 背景とプレビュー用字幕でプレビューできます。
- フォント名、サイズ、Alignment、MarginV をプリセット単位で調整できます。
- 現在の設定を新規プリセットとして保存できます。
- 選択中プリセットへ上書き保存できます。
- 選択中プリセットを消去できます。
- MP4作成中でも UI を操作し、次の素材選択、プリセット調整、次の MP4 作成を続けられます。
- Windows では ffmpeg / ffprobe 起動時に黒いコンソール画面が出ないようにしています。
- 日本語 / English の UI 切替に対応しています。

## 入力と出力

### 入力

| 種類 | 対応例 | 用途 |
| --- | --- | --- |
| 画像 | `.jpg`, `.jpeg`, `.png`, `.bmp`, `.webp` | 動画の静止画背景 |
| 音源 | `.mp3`, `.wav` | 動画の音声 |
| 字幕 | `.srt` | 焼き込み字幕。任意 |

音源は ffmpeg が読み込める形式なら実際には処理できる場合がありますが、アプリのファイル分類は主に `.mp3` と `.wav` を想定しています。

### 出力

出力は音声付き MP4 です。映像は静止画を 30fps の H.264 動画にしたものです。SRTが指定されている場合、字幕は動画に焼き込まれるため、出力後に字幕だけを非表示にすることはできません。

## 基本ワークフロー

1. アプリを起動します。
2. 左側のクイックパネルから使いたい字幕位置プリセットを選びます。
3. 画像、音源、必要ならSRTをクイックパネルへドロップします。
4. 必要なら字幕スタイルのサイズ、Alignment、MarginVを調整します。
5. 中央のプレビューで見た目を確認します。
6. 正確な確認が必要な場合は「レンダープレビュー更新」を押します。
7. 右側の「動画作成」セクションにある「MP4作成」を押します。
8. ffmpegジョブがバックグラウンドで動きます。
9. ジョブ実行中でも、次の素材を読み込んで次の動画作成を始められます。

## 起動方法

開発環境では、プロジェクトフォルダで次を実行します。

```bat
run_dev.bat
```

手動で実行する場合は次の形です。

```bat
.venv\Scripts\python.exe subtitle_image_mp4_maker.py
```

このプロジェクトでは、グローバル Python ではなく `.venv\Scripts\python.exe` の使用を前提にしています。

## ffmpeg / ffprobe の配置

アプリは `ffmpeg` と `ffprobe` を次の順序で探します。

1. アプリフォルダ直下の `bin`
   - `bin\ffmpeg.exe`
   - `bin\ffprobe.exe`
2. アプリフォルダ直下
   - `ffmpeg.exe`
   - `ffprobe.exe`
3. OS の PATH

EXE化した場合も同じ考え方です。配布フォルダ内に `bin\ffmpeg.exe` と `bin\ffprobe.exe` を置けば、PATH を変更せずに使えます。

## 素材ファイルの選択

素材は、ボタン選択またはドラッグ&ドロップで読み込めます。

### 画像

動画の背景になります。画像は ffmpeg 側で偶数サイズに丸める scale フィルタを通します。現在のアプリでは、画像の contain / cover / stretch などのフィット方式は選べません。

### 音源

MP4 の音声になります。音源を選ぶと、出力ファイル名の候補も音源ファイル名ベースで更新されます。

音源ファイルを読み込んだ場合、次の自動検出が行われます。

- 同じフォルダに同名の `.srt` があれば SRT として読み込みます。
- 音源だけをドロップした場合、同じフォルダに `<音源ファイル名>_title.png` があれば画像として読み込みます。

### SRT

指定した場合、字幕として焼き込まれます。SRT は読み込み時に UTF-8 へ正規化して作業用ファイルとして使われます。元の SRT ファイル自体は直接変更しません。SRTを指定しない場合は、字幕フィルタを使わず、静止画と音源だけでMP4を作成します。

SRTファイルを読み込んだ場合、次の自動検出が行われます。

- 同じフォルダに同名の `.mp3` または `.wav` があれば音源として読み込みます。
- 画像が明示指定されていない場合、同じフォルダに `<SRTファイル名>_title.png` があれば画像として読み込みます。

対応文字コードの推定順は次の通りです。

1. UTF-8 BOM付き
2. UTF-8
3. CP932
4. Shift_JIS

## クイックパネル

クイックパネルは、よく使うプリセットをワンクリックまたはドロップで使うための領域です。

- ページ1には通常動画用とショート動画用の主要プリセットを配置しています。
- ページ2は初期状態では空です。
- パネルへ素材をドロップすると、そのパネルのプリセットを適用し、素材も読み込みます。
- パネルにマウスオーバーすると、該当プリセットのフルネームがツールチップ表示されます。
- クイックパネルでプリセットを選ぶと、右ペインの「全プリセット」ドロップダウンも同期します。
- プリセットを消去した場合、そのプリセットを参照していたクイックパネルのスロットは空きになります。

### クイックパネル編集

「パネル編集」を押すと、クイックパネル編集用のモーダル画面が開きます。

- 移動元のスロットをクリックし、次に移動先のスロットをクリックします。
- 移動先が空きスロットの場合は移動します。
- 移動先に別のプリセットがある場合は、2つのスロットを入れ替えます。
- 選択中スロットの表示名は、同じ画面の「表示名」欄で編集できます。
- 「選択スロットを空にする」で、そのスロットの割り当てだけを削除できます。プリセット本体は削除されません。

## プリセット

プリセットは、字幕スタイルの値を保存する仕組みです。現在の実装では、主に次の値を扱います。

| 項目 | 内容 |
| --- | --- |
| fontname | ffmpeg subtitles フィルタへ渡すフォント名 |
| fontsize | ASS/libass の Fontsize |
| alignment | ASS/libass の Alignment |
| margin_v | ASS/libass の MarginV |

保存先は `presets.json` です。初期プリセットは `presets_default.json` にあります。

プリセット名とクイックパネルの表示名は、言語別に `ja` / `en` の値を持ちます。どちらか片方を `null` にできます。現在のUI言語側の値がない場合は、もう一方の言語の値へフォールバックして表示します。字幕スタイル自体は言語に関係なく同じ `style` を使います。

### 新規保存

「新規プリセットとして保存」を押すと、現在の字幕設定を新しいプリセットとして保存します。保存後、現在ページの空きスロットへ追加するか確認されます。

### 上書き保存

「選択中プリセットに上書き」を押すと、現在選択中のプリセット内容を現在の字幕設定で置き換えます。

### 消去

「選択したプリセットを消去」を押すと、現在選択中のプリセットを削除します。削除対象がクイックパネルに配置されている場合、そのスロットも空きになります。

## 字幕スタイル設定

### フォント名

ffmpeg の `subtitles` フィルタで使う `force_style` の `fontname` に反映されます。

フォント名を空または無効にすると、`force_style` に `fontname` を出しません。この場合、実際のフォント選択は libass / ffmpeg 側の既定や環境に依存します。

### サイズ

`fontsize` に反映されます。

サイズ欄の下には、次の微調整ボタンがあります。

- `-5`
- `-1`
- `+1`
- `+5`

### Alignment

現在は次の3択です。

| 表示 | 意味 |
| --- | --- |
| `6（中央上）` | 画面上側の中央 |
| `10（中央）` | 画面中央 |
| `2（中央下）` | 画面下側の中央 |

内部的には `force_style` の `alignment` に反映されます。

### MarginV

`MarginV` に反映されます。

Alignment が `6` の場合、上側からの余白として効きます。Alignment が `10` の場合は中央基準の位置調整として使われます。Alignment が `2` の場合は下側からの余白として効きます。

MarginV には次の微調整ボタンがあります。

- `-10`
- `-1`
- `+1`
- `+10`

## プレビュー

### 簡易プレビュー

簡易プレビューは Pillow / Tkinter による近似表示です。操作中の反応を速くするための表示であり、ffmpeg/libass の最終結果と完全には一致しません。

簡易プレビューでは、ASS/libass の基準解像度 `PlayResY=288` を想定し、画像の高さに合わせてフォントサイズと MarginV を換算します。さらに見た目を近づけるため、既定の補正倍率 `preview_font_scale=0.76` を使います。

### レンダープレビュー

「レンダープレビュー更新」を押すと、ffmpeg を使って1フレームだけ実際に字幕レンダリングします。最終出力に近い確認は、こちらを使ってください。

レンダープレビューでは次の処理を行います。

1. 画像が未指定なら、白い 16:9 の仮画像を作ります。
2. プレビュー字幕テキストを一時 SRT に書き出します。
3. ffmpeg の `subtitles` フィルタで字幕を焼き込みます。
4. `_last_render_preview.png` に結果を保存し、画面に表示します。

SRT が未指定の場合、プレビュー字幕欄には既定文言が入ります。日本語 UI では「プレビュー用字幕テキスト」、English UI では `Subtitle text for preview` です。

## MP4作成

「MP4作成」を押すと、その時点の入力ファイル、出力先、字幕設定をジョブとして固定します。ジョブ開始後に UI 上で素材や設定を変えても、実行中ジョブには影響しません。

MP4作成はバックグラウンドスレッドで実行されます。1本目の ffmpeg が動いている間でも、次の素材を読み込み、次の MP4 作成を開始できます。

### MP4作成に必要なもの

MP4作成には次の2つが必要です。

- 画像
- 音源

SRTは任意です。SRTが指定されている場合だけ字幕を焼き込みます。

## ffmpeg への反映

このアプリは、画面上の設定を ffmpeg / ffprobe の処理へ変換します。重要な点は次の通りです。

### 1. SRT の作業用コピー

SRTが指定されている場合、元の SRT は直接変更しません。ジョブごとの一時フォルダに `subtitle.srt` を作り、文字コードを UTF-8 に正規化します。SRTが未指定の場合、この工程はスキップします。

概念的には次の処理です。

```text
元SRT -> 一時フォルダ\subtitle.srt
```

### 2. 音源を WAV 化

音源は一度 WAV に変換します。

```text
ffmpeg -y -hide_banner -loglevel error -i <audio> -vn <work.wav>
```

この WAV は、音声長を安定して測るためと、最終エンコード時の入力を安定させるために使います。

### 3. ffprobe で音声長を測る

WAV 化した音声に対して、ffprobe で duration を取得します。

```text
ffprobe -v error -show_entries format=duration -of default=noprint_wrappers=1:nokey=1 <work.wav>
```

取得した秒数は、最終 MP4 作成時の `-t` に渡されます。これにより、音源より2～3秒長い動画ができる問題を避けます。

### 4. 字幕フィルタを作る

動画側のフィルタは次の形です。

```text
scale=trunc(iw/2)*2:trunc(ih/2)*2,subtitles='<subtitle.srt>':force_style='<style>'
```

`scale=trunc(iw/2)*2:trunc(ih/2)*2` は、入力画像の幅と高さを偶数に丸めます。H.264 / yuv420p では偶数サイズが必要になることがあるためです。

`subtitles` は ffmpeg の字幕焼き込みフィルタです。SRT を読み込み、映像へ直接焼き込みます。SRTが未指定の場合、`subtitles` フィルタと `force_style` は付けず、`scale=trunc(iw/2)*2:trunc(ih/2)*2` のみを使います。

### 5. 字幕スタイルと force_style の対応

GUI の字幕設定は、ffmpeg subtitles フィルタの `force_style` に渡されます。

| GUI項目 | force_style キー | 例 |
| --- | --- | --- |
| フォント名 | `fontname` | `fontname=Yu Gothic UI` |
| サイズ | `fontsize` | `fontsize=20` |
| Alignment | `alignment` | `alignment=6` |
| MarginV | `MarginV` | `MarginV=95` |

値が空または無効な項目は `force_style` に出力しません。

例:

```text
fontname=UD デジタル 教科書体 NK-B,fontsize=20,alignment=6,MarginV=95
```

フォント名やサイズを未指定にするプリセットでは、次のように必要な値だけが出ます。

```text
alignment=6,MarginV=90
```

### 6. 最終 MP4 作成コマンド

実際の最終出力は、概念的には次のコマンドです。

```text
ffmpeg -y -hide_banner -loglevel error ^
  -loop 1 ^
  -framerate 30 ^
  -i <image> ^
  -i <work.wav> ^
  -map 0:v ^
  -map 1:a:0 ^
  -vf <scale_and_subtitles_filter> ^
  -c:v libx264 ^
  -tune stillimage ^
  -c:a aac ^
  -ar 48000 ^
  -b:a 384k ^
  -pix_fmt yuv420p ^
  -r 30 ^
  -movflags +faststart ^
  -t <audio_duration_seconds> ^
  <output.mp4>
```

各パラメータの意味は次の通りです。

| パラメータ | 意味 |
| --- | --- |
| `-loop 1` | 静止画を連続フレームとして扱う |
| `-framerate 30` | 入力静止画のフレームレートを30fpsとして扱う |
| `-map 0:v` | 画像入力の映像ストリームを使う |
| `-map 1:a:0` | WAV入力の音声ストリームを使う |
| `-vf` | 偶数サイズ化を行う。SRT指定時のみ字幕焼き込みも行う |
| `-c:v libx264` | H.264で映像をエンコードする |
| `-tune stillimage` | 静止画動画向けに x264 を調整する |
| `-c:a aac` | AAC音声でエンコードする |
| `-ar 48000` | 音声サンプルレートを48kHzにする |
| `-b:a 384k` | 音声ビットレートを384kbpsにする |
| `-pix_fmt yuv420p` | 互換性の高いピクセル形式にする |
| `-r 30` | 出力動画を30fpsにする |
| `-movflags +faststart` | Web再生などで先頭から再生しやすくする |
| `-t` | ffprobeで測った音声長に動画長を合わせる |

### 7. レンダープレビューの ffmpeg コマンド

レンダープレビューでは1枚だけ画像を書き出します。

```text
ffmpeg -y -hide_banner -loglevel error ^
  -loop 1 ^
  -i <image_or_white_placeholder> ^
  -vf <scale_and_subtitles_filter> ^
  -frames:v 1 ^
  <preview.png>
```

MP4作成と同じ `subtitles` フィルタを使うため、簡易プレビューより正確です。

## 出力ファイル名と上書き防止

音源が指定されている場合、出力ファイル名は基本的に音源ファイル名ベースになります。

出力ファイル名は、左側のクイックパネル下にある「動画作成」セクションで確認・指定します。「指定」ボタンを押すと、新規作成する MP4 の保存先とファイル名を指定できます。

例:

```text
揺れるランプ.wav -> 揺れるランプ.mp4
```

同名の MP4 がすでにある場合、または実行中ジョブが同じ出力先を予約している場合は、次のような連番名を使います。

```text
揺れるランプ_001.mp4
揺れるランプ_002.mp4
```

PyInstaller 用のバッチファイルがあります。

```bat
build_exe_onedir.bat
```

このバッチは次の処理を行います。

- `.venv` がなければ作成する
- 依存パッケージと PyInstaller をインストールする
- `--windowed` / `--onedir` でビルドする
- `tkinterdnd2` のPythonモジュールとTcl/Tk側データを含める
- `bin\ffmpeg.exe` と `bin\ffprobe.exe` があれば配布フォルダへコピーする
- `presets_default.json`、`presets.json`、README、仕様書、CHANGELOG をコピーする

Windowsでは、アプリ内部の `subprocess.run` / `subprocess.Popen` に黒いコンソール画面を出さない設定を適用しています。

---

## トラブルシューティング

### ffmpeg または ffprobe が見つからない

`bin\ffmpeg.exe` と `bin\ffprobe.exe` があるか確認してください。なければ PATH に ffmpeg を追加するか、アプリフォルダの `bin` に配置してください。

### 字幕の見た目が簡易プレビューと違う

簡易プレビューは近似表示です。最終確認には「レンダープレビュー更新」を使ってください。

### フォントが想定と違う

ffmpeg の字幕描画は libass とシステムのフォント解決に依存します。フォント名は Windows にインストールされている実フォント名に合わせてください。フォント名を未指定にした場合、実出力では `fontname` を渡しません。

### 動画が音源より長くなる

このアプリでは、音源を一度 WAV 化し、ffprobe で測った長さを `-t` に渡します。通常は音源長に揃います。問題が出る場合は、ログに表示される音声長と元音源の状態を確認してください。

### MP4作成中に次の作業をしてよいか

可能です。MP4作成ボタンを押した時点でジョブ内容は固定されます。作成中に画面上の入力や字幕設定を変更しても、実行中ジョブには影響しません。

### 日本語SRTが文字化けする

UTF-8、CP932、Shift_JIS を順に試して読み込みます。特殊な文字コードの場合は、事前に UTF-8 へ変換してください。

---

## 制限事項

- 画像の contain / cover / stretch などのフィット方式はまだありません。
- MP4作成には、画像と音源の実ファイルが必要です。SRTは任意です。
- プレビューは画像やSRTなしでも可能です。MP4作成では画像と音源は必要です。
- 字幕スタイルUIは現時点では主要項目に絞っています。
- 出力は H.264 + AAC の MP4 固定です。
- SRT指定時の字幕は焼き込みです。ソフト字幕トラックとしては出力しません。
- ffmpeg / libass のフォント解決は環境差が出る場合があります。

## ファイル構成

代表的なファイルは次の通りです。

```text
still_to_movie_maker/
  subtitle_image_mp4_maker.py   アプリ本体
  presets_default.json          初期プリセット
  requirements.txt              Python依存パッケージ
  run_dev.bat                   開発実行用
  build_exe_onedir.bat          PyInstallerビルド用
  README.md                     GitHub表示用README
  README_ja.txt                 簡易README
  USER_GUIDE.md                 ユーザーガイド
  SPECIFICATION_ja.md           仕様メモ
  CHANGELOG.md                  変更履歴
  LICENSE                       ライセンス
  bin/
    ffmpeg.exe                  任意。ソースZIPには含めません
    ffprobe.exe                 任意。ソースZIPには含めません
```

`presets.json`、`_last_render_preview.png`、`.venv/`、`build/`、`dist/` はローカル生成物です。GitHub用ソースZIPには含めません。
